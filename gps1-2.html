Frank & Thomaz

What is pair programming?

  • Pair programming is when you have two programmers working together on the same file at the same time. The purpose is to collaborate together to bounce ideas off of each other and work more efficiently. Each programmer has a role. The most common roles are driver and navigator.

Why is pair programming important to DBC?

  • Pair programming is important to DBC because it is the way code is written in the real world. DBC wants their students to become comfortable with pair programming so that they can excel after graduation. It is easy to get stuck when programming by yourself, so by working with a partner you are more likely to overcome any obstacles you encounter.

Compare Driver and Navigator

  • The Driver is the partner that does the typing and physically writes the code. The Driver also offers feedback on the code and helps guide the Navigator.

    The Navigator is the partner who is in charge of directing what should be written. He/She instructs the driver on what to write and control what part of the code is being worked on.
